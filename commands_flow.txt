#Start AWS EC2 and generate a virtual machine of Linux window and download .pem file and copy ssh line and generate virtual machine in local computer and every time you open new terminal coonect to the virtual machine.--------------------------------------------------------------------------------------------------------------------#Download and Install Kafka in the Linux wget https://downloads.apache.org/kafka/3.3.1/kafka_2.12-3.6.1.tgztar -xvf kafka_2.12-3.6.1.tgz#Check if java is Installed or not #Install Java java -versionsudo yum install java-1.8.0-openjdkjava -versionStart Zoo-keeper:-------------------------------bin/zookeeper-server-start.sh config/zookeeper.propertiesZookeeper runs in one terminal and donot disturbOpen another window to start kafkaStart Kafka-server:----------------------------------------Duplicate the session & enter in a new console --export KAFKA_HEAP_OPTS="-Xmx256M -Xms128M"cd kafka_2.12-3.3.1bin/kafka-server-start.sh config/server.propertiesIt is pointing to private server , change server.properties so that it can run in public IP To do this , you can follow the approach shared below --Do a "sudo nano config/server.properties" - change ADVERTISED_LISTENERS to public ip of the EC2 instanceCreate the topic:-----------------------------Duplicate the session & enter in a new console --cd kafka_2.12-3.3.1bin/kafka-topics.sh --create --topic test1 --bootstrap-server {Put the Public IP of your EC2 Instance:9092} --replication-factor 1 --partitions 1Start Producer:--------------------------bin/kafka-console-producer.sh --topic test1 --bootstrap-server {Put the Public IP of your EC2 Instance:9092} Start Consumer:-------------------------Duplicate the session & enter in a new console --cd kafka_2.12-3.3.1bin/kafka-console-consumer.sh --topic test1 --bootstrap-server {Put the Public IP of your EC2 Instance:9092}
